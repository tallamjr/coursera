Now we choose a, b, c and plot the curve together with the points.

Note that by looking at where the data points lie, we can deduce some of the properties for a, b, c, as follows.

The plot must have a y-intersection that is close to 0, so c is close to 0. Also, the parabola is open downwards, so a must be negative. Finally, it has its maximum at a positive x, so b must be positive.

Although the initial attempt had the general shape of the data, we could see it resulted in values going way too negative compared to the data for larger values of x. This indicated that the value of a being -2 was too large in absolute value.

We found values of a ~ 0.8 more correctly reflected the shape of the data. This greatly improved the results, but we thought we should be able to do better. The data indicates that the peak y-value occurs near an x-value of 2. This would indicate that the derivative of the parabola should be 0 near x = 2.

The derivative for

f(x)=ax2+bx+cf(x)=ax2+bx+c
is given by
f(x)=2ax+bf(x)=2ax+b

Given that the derivative should be 0 for x ~ 2 this gives us:

04a+b04a+b
This implies
b4ab4a

So we decide to play with values of a ~ -0.8 and b ~ -4 * -0.8 = 3.2. From the data we could also see that the y-intercept was near -1 or -2, so we tried values of c in that range.

While iteratively attempting values, we also calculated and displayed the sum of absolute differences between the actual data points and what the curve would predict for those points. This provided an analytical tool for assessing the quality of the fit, so we were not simply "eye-balling" our assessment of the best fit. Note it would be possible to develop a linear program to solve exactly for the optimal values of a, b, and c to minimize the absolute differences, but that is outside the scope of our current Julia knowledge. We look forward to pursuing this at a later time.

Using our iterative method, the best fit we found was for a = -0.82, b = 2.9 and c = -1.75 resulting in a total absolute error of 7.45. Thus our best curve to fit the data is
